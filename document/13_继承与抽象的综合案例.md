某IT公司有多名员工，按照员工负责的工作不同，进行了部门的划分（研发部、维护部）。

+ 研发部(Developer)根据所需研发的内容不同，又分为 JavaEE工程师 、Android工程师 ；
+ 维护部(Maintainer)根据所需维护的内容不同，又分为 网络维护工程师(Network) 、硬件维护工程师(Hardware) 。

公司的每名员工都有他们自己的员工编号、姓名，并要做它们所负责的工作。

工作内容:

+ JavaEE工程师： 员工号为xxx的 xxx员工，正在研发电商网站
+ Android工程师：员工号为xxx的 xxx员工，正在研发电商的手机客户端软件
+ 网络维护工程师：员工号为xxx的 xxx员工，正在检查网络是否畅通
+ 硬件维护工程师：员工号为xxx的 xxx员工，正在修复电脑主板

请根据描述，完成员工体系中所有类的定义，并指定类之间的继承关系。进行XX工程师类的对象创建，完成工作方法的调用。

**注意：**

+ **实现**
    - 对于研发部（Developer）的子类JavaEE和Android，对二者的属性赋值时采用的是利用构造方法赋值
    - 对于维护部（Maintainer）的子类NetWork和Hardware，对二者的属性赋值时采用的时利用`set()`方法赋值
    - 所以在写JavaEE和Android类时，内部写了构造方法，而Network和Hardware类则不需要，这两个使用从父类的父类Employee继承而来的`set()`方法为属性赋值
+ 由于Developer和Maintainer不能重写父类Employee的`work()`方法（因为这二者都有下属的不同员工类型，且工作内容不一样），所以`Developer`和`Maintainer`也必须是抽象类

> + `3.1`注意的第七条：子类继承父类之后，需要重写父类中的所有抽象方法，不然会编译报错，除非该子类也是抽象类
>

**Employee类（抽象类）**

```java
package com.aurora.abstractDemo.Practice;

public abstract class Employee {
    private String name;
    private int id;

    public Employee() {
    }

    public Employee(String name, int id) {
        this.name = name;
        this.id = id;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getId() {
        return id;
    }

    public void setId(int id) {
        this.id = id;
    }

    public abstract void work();
}

```

**Developer类（抽象类）**

```java
package com.aurora.abstractDemo.Practice;

public abstract class Developer extends Employee{
    public Developer() {
    }

    public Developer(String name, int id) {
        super(name, id);
    }
}
```
